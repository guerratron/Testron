Introducción:

Esto puede utilizarse como método preparatorio para exámenes académicos y oposiciones al Estado. Solo se necesita el archivo de preguntas con una estructura válida. Este archivo puede estar en formato JSON (objeto o cadena) u Objeto Javascript con una variable declarada.
Esta rígida estructura puede contener parámetros de configuración (la propiedad conf) donde especificar, por ejemplo, los contenedores de la UI en los cuales embeber las preguntas y el Marcador con información y botones, o si se permiten seleccionar múltiples respuestas o una sóla, el charset a utilizar, etc.

Aquí test representa a una batería de preguntas.

Este namespace utiliza profúsamente las funciones de utilidad de un objeto interno implementado en el mismo archivo, el objeto 'util', el cual contiene métodos y objetos estáticos, ellos pueden utilizarse externamente usando su 'classpath'.
También contiene las clases Test, Test.Marker, Test.Question y Tests.Question.Answer, pero es más confortable utilizar un único método, el cual realiza todo el trabajo automáticamente:
testron.parseTestrons (test, conf)

Cargando y Utilizando el Script:

Por supuesto el primer paso será cargar el script en la zona Head de la página, por ejemplo:
 <script type="text/javascript" src="testron.min.js"> </script>

Una vez cargado, el script crea un namespace global: testron, el cual contiene todo lo necesario para construir una batería de preguntas. Esto se logra usando un simple método 'parseTestrons (test, conf)', el cual soporta dos parámetros:

  • test: Un objeto Javascript (o cadena) el cual contiene las preguntas en formato JSON. Debe ajustarse a una determinada estructura.
  • conf: Un objeto Javascript para las opciones de configuración.
Observaciones: Esta librería crea toda la UI que contiene las preguntas y una zona Marcador con un pequeño menú de botones de acción.
El Marcador debería situarse en posición fija (fixed) para una mejor visualización.

Aunque se ha hecho un esfuerzo para que la librería sea cross-browser, pueden observarse ligeras diferencias según el navegador utilizado, especialmente "IE".

GUI:

La interfaz gráfica se auto-construye y se embebe en el elemento contenedor con el 'id' pasado como parámetro en la configuración. Todo el código necesario (Javascript y HTML, CSS se ha separado para permitir maryor personalización) se encuentra definido en el propio script 'testron.js', es el responsable de visualizar dos áreas distintas, la Batería de Preguntas y la zona del Marcador:

La zona del Marcador calcula la puntuación actual y muestra otra información como el número de respuestas acertadas y fallidas, también está representada una zona de botones que simbolizan las siguientes acciones:

  • Botón Editar: Muestra un 'textarea' con la estructura JSON completa para las preguntas actuales. Esto permite la edición del texto o su copiado.
  • Botón Descargar: Permite la descarga del archivo o un copiado del texto.
  • Botón Reiniciar: Reinicia las soluciones de las preguntas y el Marcador.

El 'botón de descargar' puede fallar en determinadas configuraciones y navegadores (IE ☹). -> SOLUCIÓN: Copiar el texto 'textarea' y guardarlo en un archivo de texto.
